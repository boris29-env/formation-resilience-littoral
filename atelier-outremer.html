07

Analyse automatique d'images côtières

Extraction SIG (GeoJSON géoréférencé) par traitement d'image dans le navigateur, géoréférencement pixel→WGS84 exact, dalles « no-data » exclues. Trait de côte : segmentation mer/terre par seuillage automatique (Otsu) + mer connectée au cadre. Ouvrages : détection des portions rectilignes du trait de côte (un littoral naturel est sinueux ; une côte rectiligne signale un ouvrage longitudinal — digue/perré/enrochement) — par construction au contact de l'eau. Réglez l'emprise/détail, et accumulez toutes les extractions dans une couche cumulée unique (ou « Traiter tous les sites » pour une couverture du territoire). Détection indicative, à valider sur le terrain ; pas d'évaluation d'état ni d'altitude.

Collecte de données — consentement explicite requis
Cette plateforme peut enregistrer les transcriptions des conversations (voix du passé, trajectoires climatiques, tableau de bord), les scores de diagnostic et les votes de routage, dans le cadre de la mission inter-inspections IGEDD/IGA (réf. 014917-02). Ces données sont stockées dans Firebase, sous votre code session. Elles ne contiennent aucune information nominative et sont utilisées uniquement pour alimenter la monographie territoriale. Vous pouvez retirer votre consentement et supprimer vos données à tout moment.
● Collecte active

Données collectées

Mission : inter-inspections IGEDD/IGA, réf. 014917-02.
Finalité : alimentation de la monographie territoriale provisoire (réf. 014917-02B). Les données sont pseudonymisées (code session et groupe, sans identifiant nominatif).
Base légale : consentement explicite (vous pouvez le retirer à tout moment).
Durée de conservation : session en cours + 90 jours maximum dans Firebase.

Ce qui est collecté : transcriptions des échanges avec les personnages historiques et les trajectoires climatiques, notes du tableau de bord territorial, scores de diagnostic, votes de routage stratégique. Ce qui n'est pas collecté : nom, prénom, adresse e-mail, adresse IP, données de navigation.
